Contexte

Dans cette première mission nous allons d'abord prendre en main les données de KDou en étudiant la base de donnée et l'interrogeant sur excel.

Analyse globale de la base de données

Nous avons d'abord identifiés les clés primaires et secondaires :

  1. Clés primaires :
    1. Table Fournisseur : l’attribut « N° fournisseur »
    2. Table Produit : l’attribut « Réf produit »
    3. Table Categorie : l’attribut « Code catégorie »
    4. Table Clientele : l’attribut « Code client »
    5. Table Commande : l’attribut « N° commande »
    6. Table DetailCommande : la clé primaire par attributs d’ensemble ici est composée de l’attribut « Réf produit » et de l’attribut « N° commande ». Ces deux attributs permettent d’avoir une unicité.
  2. Clés secondaires
    1. Table Fournisseur : Aucun attribut
    2. Table Produit : les attributs « N° fournisseur » et « Code catégorie »
    3. Table Categorie : Aucun attribut
    4. Table Clientele : Aucun attribut
    5. Table Commande : l’attribut « Code client »
    6. Table DetailCommande : les attributs « Réf produit » et « N° commande »

On a donc le schéma de la BDR suivant :

FOURNISSEUR (N° fournisseur, Société, Contact, Fonction, Adresse, Ville, Région, Code postal, Pays, Téléphone)
PRODUIT (Réf produit, N° fournisseur, Code catégorie, Nom du produit, Quantité par unité, Prix unitaire, Unités en stock)
CATEGORIES (Code catégorie, Nom de catégorie, Description)
CLIENTELE (Code client, Société, Contact, Adresse, Ville, Code postal, Pays, Téléphone)
COMMANDE (N° commande, Code client, Date commande, A livrer avant, Date envoi, Port)
DETAILCOMMANDE (Réf produit, N° commande, Prix unitaire, Quantité, Remise (%))
